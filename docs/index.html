📖 Project Overview

CharithaNet is a custom deep learning system for automated COVID-19 pneumonia detection from chest X-ray images, developed as an MSc Computer Science (SLQF Level 10) research project at the Postgraduate Institute of Science (PGIS), University of Peradeniya, Sri Lanka.

The system evolved through 5 iterative experimental versions, culminating in a novel parallel-kernel CNN architecture that achieves 99.6% classification accuracy on held-out test data — with full explainability via GradCAM heat maps and Saliency Maps.

🎯 Research Objective

Design and validate a convolutional neural network capable of reliably distinguishing COVID-19-positive chest X-rays from normal/non-COVID cases, providing both accurate predictions and visual explanations that are interpretable by clinicians.

What Makes CharithaNet Different?

  • Novel Parallel-Kernel CNN: Simultaneous multi-scale feature extraction using kernels of size 3, 5, 7, and 9 in parallel branches
  • Iterative Design: 5 systematically improved experimental versions with documented trade-offs
  • Visual Explainability: GradCAM and Saliency Map visualizations highlight diagnostic regions in X-rays
  • Multi-Dataset Training: Combined IEEE COVID-CXR and Kaggle CoronaHack datasets for robustness
  • End-to-End Deployment: Flask web application for real-time inference
  • Peer-Reviewed: Published in the Colombo Journal of Multi-Disciplinary Research (CJMR), Vol. 8

✨ Methodology

📂 Dataset

Combined two publicly available chest X-ray repositories:

  • IEEE COVID Chest X-Ray Dataset
  • Kaggle CoronaHack Dataset
  • Binary labels: COVID-19 Positive / Negative
  • Train / Validation / Test splits

🧠 Parallel-Kernel CNN

CharithaNet's core innovation — a custom multi-branch CNN that extracts features at multiple spatial scales simultaneously:

  • Parallel Conv2D branches (k = 3, 5, 7, 9)
  • Concatenated feature maps
  • Dense layers with Dropout regularisation
  • Binary sigmoid output

🔬 Explainability (XAI)

Two complementary visualisation techniques to explain model decisions:

  • GradCAM — gradient-weighted class activation maps
  • Saliency Maps — pixel-level gradient attribution
  • Applied to both COVID+ and COVID− samples
  • Qualitative validation of diagnostic regions

📈 Training Strategy

Rigorous iterative experimentation across 5 versions:

  • ModelCheckpoint callbacks (best val_accuracy)
  • 25 epochs with real-time metric tracking
  • Multiple metrics: accuracy, precision, recall, F1
  • ROC-AUC, sensitivity, specificity analysis

🌐 Web Deployment

Production-ready inference via a lightweight Flask application:

  • Base64 image upload via REST API
  • OpenCV preprocessing pipeline
  • Real-time COVID-19 Positive / Negative prediction
  • Confidence score returned per request

📊 Evaluation

Comprehensive test-set evaluation beyond simple accuracy:

  • Classification report (per-class metrics)
  • Confusion matrix visualisation
  • Sensitivity (TPR) & Specificity (TNR)
  • ROC curve and AUC score

Parallel-Kernel Design

Input (224×224 grayscale chest X-ray)
        │
        ├── Conv2D (k=3, 128 filters, same, ReLU)
        ├── Conv2D (k=5, 128 filters, same, ReLU)
        ├── Conv2D (k=7, 128 filters, same, ReLU)
        └── Conv2D (k=9, 128 filters, same, ReLU)
                │
           Concatenate
                │
        MaxPooling2D  →  Flatten
                │
          Dense (256, ReLU)  →  Dropout
                │
          Dense (1, Sigmoid)
                │
     Output: COVID-19 Positive / Negative

🚀 Quick Start

Prerequisites

  • Anaconda / Miniconda installed
  • Python 3.9 (managed via Conda)
  • GPU recommended for training (CPU works for inference)

1 — Environment Setup

# Clone the repository
git clone https://github.com/LalithK90/CharithaNet.git
cd CharithaNet

# Create and activate the Conda environment
conda env create -f requirment_for_project.yml
conda activate tensorflow

2 — Run Training Notebooks

# Launch Jupyter and open any version notebook
jupyter notebook

# Notebooks (in traning-notebook/):
#   covid-19-identification-with-chest-x-ray-v1.ipynb  ← initial prototype
#   covid-19-identification-with-chest-x-ray-v2.ipynb  ← extended kernels
#   covid-19-identification-with-chest-x-ray-v3.ipynb  ← GradCAM added
#   covid-19-identification-with-chest-x-ray-v4.ipynb  ← experimental
#   covid-19-identification-with-chest-x-ray-v5.ipynb  ← final CharithaNet

3 — Run the Web App

cd charitha_net_web_app

# Install Flask dependencies
pip install -r requirements.txt

# Start the inference server
python app.py
# → http://localhost:5000

  1. Can a custom parallel-kernel CNN outperform single-kernel baselines on COVID-19 X-ray classification?
